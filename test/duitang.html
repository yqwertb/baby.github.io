                                                  # 2 介绍flex布局 有使用过什么属性？

                                                  在父元素上定义的属性：

                                                  • flex-direction 元素布局的方向 横或者竖
                                                  • flex-wrap 当元素长或宽超出父元素的尺寸是否换行
                                                  • justify-content 横向的布局方式
                                                  • align-items 竖向的布局方式
                                                  • align-content 在flex-wrap不为nowrap的情况下，多行的时候每行的布局方式

                                                  在子元素定义的属性：

                                                  • flex-grow 规定子元素的放大比例，默认值为0。

                                                    • 所有剩余的空间为1

                                                    • 只有一个元素设置flex-grow

                                                      • flex-grow小于1，将剩余空间乘以设置的值，划给这个元素
                                                      • flex-grow大于1，将所有的剩余空间划给这个元素

                                                    • 多个元素设置flex-grow

                                                      • flex-grow总和小于1，剩余空间按比例划分
                                                      • flex-grow总和大于1，剩余空间全部利用，按比例划分

                                                  • flex-shrink规定子元素的缩放比例，默认值为1 ,小于1时，元素收缩不完全，等于0时，元素不收缩。

                                                    • 只有一个元素设置flex-shrink
                                                      • flex-shrink小于1时，收缩不完全，会有内容溢出flex容器
                                                      • flex-shrink大于1时，完全收缩，完全占满flex容器
                                                    • 多个元素设置flex-shrink
                                                      • flex-shrink总和小于1时，每个元素收缩的比例就是设置的值
                                                      • flex-shrink总和大于1时，完全收缩，收缩的比例就是设置的值

                                                  • flex-basis 定义了在分配剩余空间之前元素的默认大小，默认值为auto

                                                  • order 值越小，位置越靠后

                                                  • align-self 规定当前元素和其他元素不一样的布局方式，覆盖align-items的设置

                                                  em： 根据离自己最近元素的字体大小计算，如果本身设置字体大小，则继承本身的，反之向上寻找。

                                                  # *7 HTTP状态码

                                                  1. 1xx：服务器已经收到请求
                                                  2. 2xx：响应成功
                                                  3. 3xx：需要进一步的操作，即重定向
                                                  4. 4xx：客户端请求错误
                                                  5. 5xx：服务器响应错误

                                                  常见状态码

                                                  101 协议升级

                                                  204 NO Content

                                                  ​ 用来做正式请求的预请求，只需要确认确认请求能不能通过，只需要一个结果，不要返回其他内容

                                                  206 Partial Content

                                                  只返回了请求资源的部分内容，需要在请求头中加入range

                                                  301 Moved Permanently

                                                  资源被永久重定向，响应头部的location返回一个新地址

                                                  302 Found

                                                  资源被临时重定向

                                                  303

                                                  强制将POST请求转为GET请求

                                                  304 Not Modifed

                                                  协商缓存

                                                  307 hsts跳转

                                                  在服务器响应头添加Strict-Transport-Security

                                                  400 Bad Request

                                                  请求中语法错误

                                                  401 Unauthorized

                                                  如果请求头中的Authorization内容有问题，即未通过认证

                                                  403 Forbidden

                                                  无权限访问

                                                  500 Internal Server Error

                                                  服务器故障

                                                  # 8 GET和POST的区别

                                                  1. 浏览器在回退的时候，get不会重新请求，post会重新请求
                                                  2. get请求会被浏览器主动缓存和保留在浏览器的历史记录里，post不会
                                                  3. 浏览器对get请求中的参数有大小限制，post没有
                                                  4. get的参数直接跟在url后面，post是在请求体中
                                                  5. get请求对数据类型有限制，post请求没有

                                                  # 9 浏览器事件循环机制

                                                  浏览器事件循环：js是单线程的，代码执行的过程中会产生执行上下文，执行上下文会被按顺序压入主执行栈中，遇到宏任务和微任务会被加入到相应的任务队列中。当主执行栈为空后，按顺序执行任务队列，先执行微任务队列，再执行宏任务队列。如果任务队列中发现宏任务和微任务，再加入到相应的任务队列中。

                                                  • 宏任务
                                                    • setTimeout
                                                    • setImmediate
                                                    • setInterval
                                                    • script
                                                  • 微任务
                                                    • promise
                                                    • process.nextTick
                                                    • MutationObserver

                                                  # 10 JS数据类型（深拷贝、浅拷贝、类型检测）

                                                  • 基本

                                                    undefined null Number String Boolean symbol

                                                  • 引用

                                                    object array function Math RegExp

                                                  • 基本数据类型和引用数据类型有什么区别

                                                    • 基本类型

                                                      • 存储在栈内存中
                                                      • 按值访问，操作实际保存的值
                                                      • 复制变量，改变源数据不会影响新变量，会在栈中创建新的内存存储变量

                                                    • 引用数据类型

                                                      • 存储在堆内存中
                                                      • 按引用访问，变量存的是地址
                                                      • 复制变量是复制地址，改变任意一个数据会影响另一个数据

                                                  • 两种类型拷贝有什么区别？

                                                    引用类型数据的赋值不是浅拷贝

                                                    浅拷贝只拷贝第一层的基本数据类型的值和引用数据类型的地址

                                                    • 赋值：基本类型的赋值，两个变量互不影响；引用类型中的基本类型会影响
                                                    • 浅拷贝：创建一个新的对象，属性是基本类型的值就拷贝数据，引用类型就拷贝地址
                                                    • 深拷贝：创建一个新的对象，使引用类型的属性和源数据不互相影响。

                                                    浅拷贝：object.assign() array.concat() array.slice()

                                                    和原数据是否指向同一地址 第一层数据为基本数据类型 第一层数据为引用数据类型
                                                    赋值 是 原数据改变 原数据改变
                                                    浅拷贝 否 原数据不改变 原数据改变
                                                    深拷贝 否 原数据不改变 原数据不改变

                                                  • 递归的方式实现深拷贝 有什么问题

                                                    当数据的层次很深，可能造成栈溢出

                                                  # 14 var let const区别

                                                  var 不存在块级作用域，在变量赋值之前使用变量，变量值是undefined

                                                  let 存在块级作用域，只能在声明变量后使用

                                                  const 存在块级作用域，对于基本数据类型，不能修改变量值；对于引用数据类型，可以改变变量中的属性

                                                  # 16 event bus

                                                  在main.js中给Vue的原型上添加一个新的vue实例（不具有DOM组件，只有Vue的方法）

                                                  Vue.prototype.$eventBus = new Vue()

                                                  发送事件：

                                                  `import eventBus from './main,js'

                                                  mouted() {

                                                  ​ eventBus.$emit('事件名',数据)

                                                  }`

                                                  接受事件：

                                                  `import eventBus from './main,js'

                                                  mouted() {

                                                  ​ eventBus.$on('事件名',callback(payload))

                                                  }` 销毁事件监听

                                                  • EventBus.$off('事件名', callback)，只移除这个回调的监听器。
                                                  • EventBus.$off('事件名')，移除该事件所有的监听器。
                                                  • EventBus.$off()， 移除所有的事件监听器，注意不需要添加任何参数。
