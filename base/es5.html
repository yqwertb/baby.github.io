# Event Loop

​ JS是单线程的，在代码执行过程中会产生执行上下文，这些执行上下文会被按顺序执行栈中，如果遇到异步操作，将异步操作压入任务队列，任务队列又分为宏任务和微任务两个队列。当执行栈中为空后，开始按顺序处理任务队列中的任务，先处理微任务再处理宏任务，把它们放入执行栈中。

​ 当执行宏任务时如果再有宏任务和微任务，按之前的操作，加入宏任务或者微任务队列。当之前的微任务和宏任务处理完后，再处理新加入任务队列的任务。

# onclick和addEventListener的区别

onclick属于Inline events，addEventListener属于Event Listeners。

Inline events可以在标签中使用也可以在js中使用（ele.onclick = function(){}），并且一个元素只能绑定一个事件和回调函数，因为后面的会覆盖前面的。

Event Listeners有addEventListener（主流浏览器）和attachEvent（IE浏览器），可以给一个元素绑定多个事件和回调函数。可以取消事件的绑定。

# all() apply() bind()

三者都是改变函数执行时的上下文 即this的指向

call()接受两个或多个参数，第一个是this的指向，其他为原函数接受的参数

apply()接受两个参数，第一个是this的指向，第二个为一个数组，存入原函数接受的参数

call()和apply()执行后，原函数立即执行，且只是临时改变this指向一次

bind()第一个参数是this的指向，后面传入一个参数列表，可以分多次传入，改变this指向后不会立即执行，而是返回一个永久改变this指向的函数

# JS的事件模型

# 事件分为三个阶段

  • 捕获阶段：从DOM的顶层一直到事件发生的元素上
  • 目标阶段：事件流传播到事件发生的元素上
  • 冒泡阶段：从事件发生的元素向上传播

# 事件模型

  • 原始事件模型（DOM0）

    // 直接在html中绑定
<input type="button" onclick="fun()">
    
//在js中获取dom节点进行绑定
var btn = document.getElementById('.btn');
btn.onclick = fun;

    DOM0级事件只支持冒泡，不支持捕获

    同一类型的事件只能绑定一个回调函数

  • 标准事件模型（DOM2）

    addEventListener(eventType, handler, useCapture)

    useCapture表示是否在捕获阶段进行处理，默认值为false，即在冒泡阶段进行处理。

# 几个方法

event.stopPropagation() 冒泡机制下，阻止事件进一步冒泡

event.preventDefault() 取消事件的默认行为

# parseInt()

如果字符串中的字符不是指定radix的数字，将忽略该字符及之后所有的字符。

如果字符串以'0x'开头，radix被指定为16

如果字符串以'0'开头，radix被指定为8或10

# DOM事件

e.stopPropagation()阻止事件传播，不论是冒泡还是捕获

addEventListener()的第三个参数是是否使用捕获的意思，默认是否

先会把所有的事件回调函数绑定到dom节点上，按照先捕获后冒泡的顺序，遇到设置捕获触发的事件，进行回调。

# 两个dom节点寻找最近的共同父节点

使用dom的方法contains()

function commonParentNode(oNode1, oNode2) {
    if(oNode1.contains(oNode2)){
        return oNode1;
    }else{
        return commonParentNode(oNode1.parentNode,oNode2);
    }
}

用set存储

function commonParentNode(oNode1, oNode2) {
    if(oNode1===oNode2) return oNode1;
    let s=new Set();
    while(!s.has(oNode1)&&!s.has(oNode2)){
         s.add(oNode1);
         s.add(oNode2);
         if(oNode1.parentNode) oNode1=oNode1.parentNode;
         if(oNode2.parentNode) oNode2=oNode2.parentNode;
    }
    return s.has(oNode1)?oNode1:oNode2;
}
